Features

  1. Пользователь авторизуется на страничке через SSO
  2. Username берется из e-mail из данных SSO и передается сервису в качестве аргумента
  3. Пользователя создается в БД
  4. Проверяется наличие записи в таблице ldap_accounts
    • Существует - отображает пользователю параметры подключения
    • Не существует - создает новый аккаунт
  5. Сервис идет в AD и для текущего пользователя
    • Если аккаунта в AD не существует
      • Генерирует пароль
      • Создает аккаунт, который равен username пользователя
      • Создает OU
      • Добавляет пользователя в группу
      • Создает запись в БД
    • Если аккаунт существует
      • Генерирует новый пароль и принудительно сбрасывает его
  6. Записывает данные в БД, где этот аккаунт привязан к ранее созданному пользователю, чтобы не менять пароль каждый раз при входе на страничку сервиса, т.к. в AD нельзя просто так посмотреть пароль, можно только сбросить.
  7. Показыват пользователю таблицу со значениями параметров подключения
